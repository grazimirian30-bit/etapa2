Módulo 4: Arquivos e TAD

Manipulação de Dados Externos e Organização de Projetos

1. Fluxo de Trabalho com Arquivos

Para manipular arquivos em C, usamos o ponteiro especial do tipo FILE *. O ciclo de vida é sempre: Abrir ➔ Processar ➔ Fechar.

fopen()
Abrir
➔
fprintf() / fscanf()
Processar
➔
fclose()
Fechar

Modos de Abertura Principais

Modo Descrição Comportamento
"r" Read (Leitura) O arquivo deve existir.
"w" Write (Escrita) Cria um novo ou apaga o conteúdo existente.
"a" Append (Anexar) Adiciona conteúdo ao final do arquivo.
"r+" Leitura/Escrita Permite ler e modificar um arquivo existente.

2. Lendo e Escrevendo Dados

Existem funções específicas para cada tipo de dado (texto, caracteres ou blocos binários).

FILE *arq;
arq = fopen("dados.txt", "w");

if (arq == NULL) {
    printf("Erro ao abrir arquivo!");
    return 1;
}

fprintf(arq, "ID: %d | Nome: %s", id, nome); // Escreve no arquivo
fclose(arq); // Libera o recurso
💡 Verificação de Segurança: Sempre verifique se o ponteiro FILE * é NULL após o fopen. Tentar escrever em um arquivo que não abriu causará erro crítico.

3. Tipos Abstratos de Dados (TAD)

O TAD é uma técnica de programação onde se separa a interface (o que o módulo faz) da implementação (como ele faz).

Estrutura de um Projeto TAD:

  • Arquivo .h (Header): Protótipos das funções e definições de tipos. É o que o "usuário" do seu código vê.
  • Arquivo .c (Implementação): Onde o código das funções realmente está escrito.
  • Main.c: Onde você apenas consome as funções do seu TAD.
Vantagem: Facilita o trabalho em equipe e a manutenção. Você pode mudar a lógica interna sem quebrar quem usa sua biblioteca.

4. Arquivos Binários (fwrite/fread)

Ao contrário dos arquivos de texto, os binários salvam a struct inteira de uma vez, exatamente como ela está na memória RAM.

// Escrevendo uma struct inteira
fwrite(&aluno, sizeof(struct Aluno), 1, arq);

// Lendo uma struct inteira
fread(&aluno, sizeof(struct Aluno), 1, arq);

É o método mais rápido e eficiente para salvar grandes volumes de dados complexos.

Views, Transações, Funções e Procedures (T-SQL)

🖼️ 1. Views: Tabelas Virtuais

Uma View é como uma consulta salva. Ela não ocupa espaço com dados duplicados, apenas "lembra" do SELECT complexo que você criou.

Vantagens:
  • Segurança: Esconde colunas sensíveis (como salários).
  • Simplicidade: Transforma um JOIN de 5 tabelas em um "SELECT * FROM View".
CREATE VIEW v_TotalVendas AS
SELECT Nome, SUM(Total) FROM Vendas
GROUP BY Nome;

🛡️ 2. Transações: O Tudo ou Nada

Garante a integridade. Se você transfere dinheiro, o banco precisa tirar de uma conta E colocar na outra. Se uma falhar, o sistema desfaz tudo.

BEGIN
Início
→
COMMIT
Sucesso (Salva)

BEGIN TRANSACTION;
   UPDATE Conta SET Saldo -= 100 WHERE ID=1;
   UPDATE Conta SET Saldo += 100 WHERE ID=2;
IF (@@ERROR <> 0) ROLLBACK;

Questão 1

Em bancos de dados relacionais, qual é a principal finalidade de uma VIEW?

✅ Resposta Correta: C

Explicação Geral

Uma View é uma tabela virtual criada a partir de uma consulta SQL. Ela não armazena dados fisicamente, mas apresenta informações obtidas de uma ou mais tabelas.

Exemplo


CREATE VIEW VW_CLIENTES AS

SELECT
NOME,
CPF

FROM CLIENTE;

Aplicação Profissional

Views são utilizadas para simplificar consultas complexas e restringir o acesso direto às tabelas.

Questão 2

Qual é o principal objetivo de uma transação em banco de dados?

✅ Resposta Correta: B

Uma transação garante que um conjunto de operações seja executado completamente ou não seja executado. 


BEGIN TRANSACTION

UPDATE CONTA
SET SALDO = SALDO - 500

UPDATE CONTA
SET SALDO = SALDO + 500

COMMIT

Se ocorrer um erro, utiliza-se ROLLBACK para desfazer todas as alterações.

Questão Discursiva 1

Explique o conceito de View em bancos de dados relacionais e apresente suas principais vantagens.

Resposta Esperada

Uma View é uma tabela virtual criada a partir de uma consulta SQL. Ela não armazena dados fisicamente, mas apresenta informações obtidas de uma ou mais tabelas existentes.

Principais Vantagens

  • Simplificação de consultas complexas.
  • Maior segurança dos dados.
  • Reutilização de consultas.
  • Facilidade de manutenção.
  • Controle de acesso às informações.

Exemplo


CREATE VIEW VW_CLIENTES

AS

SELECT
NOME,
CPF

FROM CLIENTE;

Aplicação Profissional

Uma empresa pode criar Views específicas para departamentos diferentes, permitindo que cada setor visualize apenas os dados necessários para suas atividades.

Questão Discursiva 2

Explique o conceito de Transação em banco de dados e descreva sua importância para a integridade das informações.

Resposta Esperada

Uma transação é um conjunto de operações executadas como uma única unidade lógica de trabalho. Todas as operações devem ser concluídas com sucesso ou todas devem ser desfeitas.

Propriedades ACID

  • Atomicidade: tudo ou nada.
  • Consistência: preservação das regras do banco.
  • Isolamento: transações independentes.
  • Durabilidade: alterações permanentes após confirmação.

Exemplo

Uma transferência bancária envolve débito em uma conta e crédito em outra. Se uma etapa falhar, toda a operação deve ser cancelada.

Questão Discursiva 3

Explique a diferença entre Stored Procedures e Functions no SQL Server.

Resposta Esperada

Stored Procedures são conjuntos de comandos SQL armazenados no banco de dados para execução de tarefas específicas. Functions também armazenam lógica SQL, porém obrigatoriamente retornam um valor.

Diferenças Principais

Stored Procedure Function
Pode executar várias operações. Retorna obrigatoriamente um valor.
Pode alterar dados. Normalmente utilizada para cálculos.
Executada com EXEC. Pode ser utilizada em consultas SELECT.

Questão Discursiva 4

Descreva as principais funções de agregação disponíveis na linguagem SQL e apresente exemplos de utilização.

Resposta Esperada

As funções de agregação são utilizadas para realizar cálculos sobre conjuntos de registros.

Principais Funções

  • SUM() → Soma valores.
  • AVG() → Calcula média.
  • COUNT() → Conta registros.
  • MAX() → Retorna maior valor.
  • MIN() → Retorna menor valor.

Exemplo


SELECT AVG(SALARIO)
FROM FUNCIONARIO;

SELECT COUNT(*)
FROM CLIENTE;

Aplicação

São amplamente utilizadas em relatórios gerenciais e análises estatísticas.

Questão Discursiva 5

Uma empresa deseja automatizar o cálculo de comissão dos vendedores utilizando SQL Server. Explique qual recurso seria mais adequado entre View, Function e Stored Procedure e justifique sua resposta.

Resposta Esperada

A melhor opção seria uma Function quando o objetivo for apenas calcular e retornar um valor. Caso seja necessário realizar várias operações, atualizações ou registros adicionais, uma Stored Procedure pode ser mais adequada.

Exemplo de Function


CREATE FUNCTION FN_COMISSAO

(@VALOR MONEY)

RETURNS MONEY

AS

BEGIN

RETURN @VALOR * 0.10

END

Justificativa

Functions são ideais para cálculos reutilizáveis e podem ser utilizadas diretamente em consultas SQL.

Conclusão

A escolha depende do objetivo da automação, mas para cálculos simples e reutilizáveis a Function é normalmente a solução mais eficiente.

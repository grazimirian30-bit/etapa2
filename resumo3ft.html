Structs, Heap Memory e Alocação de Baixo Nível

1. Structs e a Organização de Dados

As Structs (Registros) permitem criar novos tipos que agrupam diferentes variáveis. Elas são a base para estruturas como Listas Ligadas e Árvores.

typedef struct {
    char nome[50];
    int idade;
    float salario;
} Funcionario;

Funcionario f1; // Criando uma instância

O Operador de Ponteiro Struct (->)

Quando acessamos uma struct através de um ponteiro, usamos a "seta" (->), que faz a desreferência automática.

Funcionario *ptr = &f1;

ptr->idade é o mesmo que (*ptr).idade

2. Alocação Dinâmica na Heap

Diferente da Stack (automática), a Heap é uma memória manual que não se apaga sozinha ao fim de uma função.

Função Objetivo Principal Estado da Memória
malloc() Aloca tamanho em bytes. Contém lixo de memória.
calloc() Aloca N elementos de X tamanho. Memória limpa (tudo zero).
realloc() Aumenta ou diminui o bloco. Mantém os dados anteriores.

4. Perigos de Memória

Memory Leak

Ocorre quando você perde o endereço da memória alocada sem dar free(). O programa continua ocupando RAM inutilmente.

Dangling Pointer

Ocorre quando você acessa uma memória que já foi liberada com free(). Causa comportamentos imprevisíveis.

Boa Prática: Após o free(p), faça sempre p = NULL;.

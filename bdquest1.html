📚 Questionário Comentado - Banco de Dados (Encontro 1)

Questão 1

Qual é o principal objetivo da Análise de Requisitos em um projeto de Banco de Dados?

A) Criar tabelas físicas do banco
B) Definir consultas SQL
C) Coletar informações necessárias para representar o negócio
D) Configurar o servidor do banco

✅ Gabarito: C

A análise de requisitos é a etapa responsável por compreender o negócio, levantando entidades, atributos e relacionamentos que serão utilizados na construção do banco de dados.

Questão 2

Durante a Análise de Requisitos são identificados principalmente:

A) Índices e Procedures
B) Entidades, atributos e relacionamentos
C) Servidores e usuários
D) Tipos de backup

✅ Gabarito: B

Nessa fase ocorre a definição dos elementos fundamentais do banco: entidades, atributos e relacionamentos.

Questão 3

Qual é a sequência correta dos níveis de modelagem de dados?

A) Conceitual → Lógico → Físico
B) Lógico → Conceitual → Físico
C) Físico → Conceitual → Lógico
D) Conceitual → Físico → Lógico

✅ Gabarito: A

Primeiro representamos o negócio (conceitual), depois adaptamos ao SGBD (lógico) e por fim implementamos a estrutura física.

Questão 4

O Modelo Conceitual tem como principal finalidade:

A) Definir comandos SQL
B) Criar tabelas físicas
C) Simplificar a visão do mundo real usando dados e relacionamentos
D) Configurar usuários do banco

✅ Gabarito: C

O modelo conceitual representa apenas os elementos importantes do negócio, sem preocupação com tecnologia ou SGBD.

Questão 5

No exemplo do cadastro de produtos apresentado pelo professor, qual item é considerado um atributo?

A) Produto
B) Categoria
C) Loja
D) Sistema

✅ Gabarito: B

Categoria é uma característica do produto, logo é considerada um atributo da entidade Produto.

Questão 6

O Modelo Lógico é responsável por:

A) Representar apenas o negócio
B) Definir a estrutura lógica dos dados conforme o SGBD
C) Criar backups
D) Configurar usuários

✅ Gabarito: B

O Modelo Lógico adapta a modelagem para o SGBD escolhido, definindo tipos de dados e estruturas.

Questão 7

Qual característica é definida completamente no Modelo Lógico?

A) Relatórios
B) Processos de negócio
C) Tipos de dados
D) Hardware

✅ Gabarito: C

No modelo lógico os tipos de dados já são especificados.

Questão 8

O Modelo Físico detalha:

A) Regras de negócio
B) Tabelas, campos e tipos de dados
C) Fluxogramas
D) Algoritmos

✅ Gabarito: B

O modelo físico é a etapa que prepara a implementação real do banco.

Questão 9

Uma entidade representa:

A) Um comando SQL
B) Um conceito ou objeto do negócio
C) Um atributo
D) Um relacionamento

✅ Gabarito: B

Entidades representam elementos importantes do negócio, como Cliente, Produto ou Funcionário.

Questão 10

Cada ocorrência individual de uma entidade recebe o nome de:

A) Relacionamento
B) Cardinalidade
C) Instância de Entidade
D) Chave Estrangeira

✅ Gabarito: C

Uma instância é um objeto específico pertencente à entidade.

Questão 11

Qual atributo não pode ser subdividido?

A) Composto
B) Multivalorado
C) Simples
D) Primário

✅ Gabarito: C

Atributos simples são indivisíveis.

Questão 12

O atributo Endereço composto por Rua, Número e Cidade é:

A) Simples
B) Primário
C) Composto
D) Multivalorado

✅ Gabarito: C

Atributos compostos podem ser divididos em partes menores.

Questão 13

Qual atributo pode possuir vários valores para um mesmo registro?

A) Simples
B) Composto
C) Multivalorado
D) Primário

✅ Gabarito: C

Exemplo: vários telefones para uma mesma pessoa.

Questão 14

Qual atributo identifica unicamente uma instância?

A) Simples
B) Primário
C) Composto
D) Multivalorado

✅ Gabarito: B

O atributo primário não admite valores repetidos.

Questão 15

Os relacionamentos servem para:

A) Conectar entidades
B) Criar tabelas
C) Definir atributos
D) Criar backups

✅ Gabarito: A

Relacionamentos estabelecem ligações entre entidades.

Questão 16

Um relacionamento envolvendo apenas uma entidade é chamado de:

A) Binário
B) Ternário
C) Unário
D) Associativo

✅ Gabarito: C

Também chamado de relacionamento recursivo.

Questão 17

Um relacionamento entre Funcionário e Setor é:

A) Unário
B) Binário
C) Ternário
D) Associativo

✅ Gabarito: B

Participam duas entidades.

Questão 18

A cardinalidade mínima representa:

A) Número obrigatório mínimo de participações
B) Número máximo de participações
C) Número de atributos
D) Número de tabelas

✅ Gabarito: A

Ela indica a participação obrigatória em um relacionamento.

Questão 19

Qual tipo de especialização obriga toda entidade a pertencer a apenas uma subentidade?

A) Parcial Compartilhada
B) Total Compartilhada
C) Parcial Disjuntiva
D) Total Disjuntiva

✅ Gabarito: D

Toda ocorrência deve participar de uma única especialização.

Questão 20

Uma Entidade Associativa normalmente surge em relacionamentos:

A) 1:1
B) 1:N
C) N:N
D) Unários

✅ Gabarito: C

A entidade associativa resolve relacionamentos muitos-para-muitos, permitindo armazenar informações adicionais.

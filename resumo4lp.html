💾 1. Persistência de Dados em Larga Escala

A persistência é o mecanismo que permite que o estado de um aplicativo sobreviva ao encerramento do processo. Enquanto a Memória Volátil (RAM) é rápida mas temporária, o Armazenamento Secundário (Disco) é permanente. Na Unidade 4, aprendemos que um programador profissional não apenas processa dados, ele os governa através do tempo.

1.1 O Handle de Arquivo (Ponteiro)

Ao abrir um arquivo em Python, você cria um objeto de arquivo. Este objeto mantém um "ponteiro" que rastreia exatamente onde você está lendo ou escrevendo. Se você ler 10 caracteres, o ponteiro avança 10 posições, esperando a próxima instrução [80-81].

📂 2. Operações de Entrada e Saída (I/O)

A função open() é o portal de entrada. Mas a segurança do dado depende do Modo de Abertura escolhido. O uso incorreto do modo pode causar a perda irreversível de bancos de dados inteiros.

ModoOperação TécnicaImpacto no Arquivo Existente
'r'Read (Somente Leitura)Nenhum. Falha se o arquivo não existir.
'w'Write (Escrita Destrutiva)Truncagem Automática: Apaga 100% do conteúdo ao abrir.
'a'Append (Anexação)Preserva os dados. Posiciona o ponteiro no final do arquivo.
'r+'Update (Leitura/Escrita)Permite sobrescrever partes específicas sem apagar tudo.
'b'Binary (Binário)Usado para imagens, áudios ou arquivos executáveis (não-texto).

2.1 Métodos de Leitura Profunda [82-84]

  • read(): Carrega o arquivo INTEIRO para a RAM. Cuidado: arquivos de 10GB podem travar o sistema.
  • readline(): Lê apenas uma linha por vez. Eficiente em termos de memória.
  • readlines(): Transforma cada linha do arquivo em um item de uma lista Python.
# O Padrão Ouro: Gerenciador de Contexto (with)
with open("usuarios.log", "a", encoding="utf-8") as arquivo:
    linha = "Registro de acesso: 2023-10-27\n"
    arquivo.write(linha)
# Aqui o arquivo é FECHADO automaticamente, mesmo que ocorra um erro!

🛡️ 3. Tratamento de Exceções e Robustez

Erros são inevitáveis em sistemas que interagem com o mundo externo. O tratamento de exceções é a prática de prever falhas e fornecer um caminho de recuperação gracioso, em vez de permitir o encerramento abrupto (crash) [85-88].

⚠️ Diferença Crítica: Erro de Sintaxe vs. Exceção
Erros de sintaxe são falhas na escrita do código (detectados antes de rodar). Exceções são erros lógicos que ocorrem DURANTE a execução (arquivo não encontrado, divisão por zero, rede fora do ar).

3.1 O Bloco Try-Except-Else-Finally

Try Zona de Risco: Contém o código que pode gerar uma exceção.
Except Plano de Contingência: Executado apenas se o erro ocorrer.
Else Caminho de Sucesso: Executado apenas se NÃO houver erro no Try.
Finally Cláusula de Garantia: Executa SEMPRE. Usado para limpar memória ou fechar conexões. 
try:
    f = open("config.json", "r")
except FileNotFoundError:
    print("Configuração não encontrada! Usando padrões...")
except Exception as e:
    print(f"Erro inesperado: {e}")
else:
    print("Configurações carregadas com sucesso.")
finally:
    print("Operação de I/O finalizada.")

🧩 4. Estratégias de Solução de Problemas

A programação é 10% escrita de código e 90% raciocínio lógico. A Unidade 4 conclui ensinando como decompor problemas complexos em instruções atômicas [90-95].

O Método Científico na Programação:
  1. Entender: Quais são as entradas? Qual a saída esperada?
  2. Planejar: Desenhar o algoritmo (fluxograma ou pseudocódigo).
  3. Dividir: Quebrar o problema em funções menores (Modularização da Unidade 3).
  4. Implementar: Escrever o código Python.
  5. Validar: Realizar testes unitários e tratar exceções.

Depuração (Debugging)

O processo de encontrar a causa raiz de um comportamento inesperado. Ferramentas como o Debugger da IDLE ou do VS Code permitem pausar o tempo e inspecionar o valor de cada variável em tempo real, evitando o "chute" na hora de consertar bugs.

Questão 1

Uma empresa deseja registrar permanentemente os dados de vendas realizadas durante o dia. Caso o computador seja desligado, os dados precisam permanecer disponíveis para consultas futuras. Qual recurso deve ser utilizado?

✅ Resposta Correta: D

Arquivos permitem armazenar dados de forma permanente em dispositivos como SSDs e HDs. Diferentemente da memória RAM, os dados permanecem disponíveis mesmo após o encerramento do programa ou desligamento do computador.

Questão 2

Analise o código: 

arquivo = open("dados.txt","w")
arquivo.write("Python")
arquivo.close()

Qual é o principal objetivo da instrução "w"?

✅ Resposta Correta: B

O modo de abertura "w" permite gravar informações em um arquivo. Caso ele já exista, seu conteúdo anterior será substituído pelo novo conteúdo gravado.

Questão 3

Observe o código: 

arquivo = open("dados.txt","a")
arquivo.write("Novo Registro")
arquivo.close()

Qual será o comportamento do programa?

✅ Resposta Correta: C

O modo append ("a") preserva os dados existentes e adiciona novas informações ao final do arquivo. É muito utilizado para registros históricos e logs.

Questão Discursiva 1

Explique a diferença entre armazenamento em memória RAM e armazenamento em arquivos permanentes. Cite uma situação prática em que o uso de arquivos seja indispensável.

✅ Resposta Esperada

A memória RAM é um armazenamento temporário utilizado durante a execução dos programas. Quando o computador é desligado, todas as informações presentes nela são perdidas. Os arquivos permanentes ficam armazenados em dispositivos como HDs e SSDs, permanecendo disponíveis mesmo após o desligamento do equipamento. Exemplos: cadastro de clientes, histórico escolar, sistema bancário, prontuários médicos e registros de vendas.

Questão Discursiva 2

Descreva as diferenças entre os modos de abertura de arquivos abaixo: 

r
w
a

Explique quando cada um deles deve ser utilizado.

✅ Resposta Esperada

O modo r é utilizado para leitura do conteúdo existente. O modo w é utilizado para gravação, criando ou sobrescrevendo completamente um arquivo. O modo a adiciona novas informações ao final do arquivo sem apagar os dados já existentes. Cada modo deve ser escolhido de acordo com a necessidade da aplicação.

Questão Discursiva 3

Analise o código abaixo e explique detalhadamente o que acontece durante sua execução. 

arquivo = open("notas.txt","w")
arquivo.write("8.5")
arquivo.close()

✅ Resposta Esperada

O programa abre ou cria o arquivo chamado "notas.txt" utilizando o modo de escrita. Em seguida grava o valor "8.5" dentro do arquivo. Por fim executa o método close(), encerrando o acesso ao arquivo e liberando os recursos utilizados pelo sistema operacional.

Questão Discursiva 5

Uma universidade precisa armazenar notas, matrículas, frequência e histórico dos alunos durante vários anos. Explique por que utilizar apenas variáveis durante a execução do programa não seria suficiente para atender esse requisito.

✅ Resposta Esperada

As variáveis armazenam informações apenas enquanto o programa está em execução. Quando o sistema é encerrado, os dados presentes na memória são perdidos. Por isso é necessário utilizar arquivos ou bancos de dados para garantir armazenamento permanente, permitindo consultas e atualizações futuras dos registros acadêmicos.

Questão Discursiva 6

Explique a importância do comando close() após a manipulação de arquivos e discuta os possíveis impactos de sua ausência em sistemas de grande porte.

✅ Resposta Esperada

O comando close() encerra o acesso ao arquivo e libera os recursos utilizados pelo sistema operacional. Sua ausência pode provocar desperdício de memória, bloqueio de arquivos, falhas de gravação e redução do desempenho do sistema. Em aplicações corporativas, o fechamento adequado dos arquivos é essencial para garantir integridade dos dados e estabilidade da aplicação.

Questão Discursiva 7

Analise a importância do tratamento de exceções durante operações de leitura e gravação de arquivos. Por que aplicações profissionais utilizam estruturas de tratamento de erros?

✅ Resposta Esperada

O tratamento de exceções permite que o sistema lide adequadamente com situações inesperadas, como arquivos inexistentes, permissões insuficientes ou dados corrompidos. Sem esse tratamento, a aplicação pode encerrar abruptamente. Sistemas profissionais utilizam exceções para garantir robustez, confiabilidade e melhor experiência para o usuário.

Questão Discursiva 8

Uma empresa armazena diariamente milhares de registros de vendas. Explique como a manipulação de arquivos contribui para auditorias, relatórios gerenciais e tomada de decisões estratégicas.

A manipulação de arquivos permite armazenar e recuperar grandes volumes de dados históricos. Essas informações podem ser utilizadas para gerar relatórios, realizar auditorias, identificar tendências e apoiar processos de tomada de decisão. Sem persistência de dados, análises estratégicas se tornariam inviáveis.

Questão Discursiva 9

Explique a diferença entre erros de sintaxe e exceções em programação. Apresente exemplos de situações em que cada um deles pode ocorrer.

✅ Resposta Esperada

Erros de sintaxe ocorrem quando o código viola as regras da linguagem de programação, impedindo sua execução. Exceções ocorrem durante a execução do programa, mesmo que a sintaxe esteja correta. Exemplos incluem tentativa de abrir um arquivo inexistente, divisão por zero ou falhas de conexão.

🖼️ 1. Views: Abstração e Segurança Técnica

Uma View não é apenas um "atalho". No SQL Server, ela funciona como uma camada de segurança e simplificação da arquitetura.

Por que usar Views além do óbvio?

Independência de Dados Se você mudar o nome de uma coluna na tabela física, basta atualizar a View e as aplicações que a usam não quebrarão.
Filtros de Segurança Você pode criar uma View que filtra WHERE Departamento = 'Vendas' e dar permissão de acesso apenas a ela, impedindo que o usuário veja dados de outros setores. 
-- Criando uma View que une 3 tabelas e mascara dados
CREATE VIEW vw_RelatorioVendas AS
SELECT 
    C.Nome AS Cliente,
    P.Descricao AS Produto,
    V.DataVenda,
    (V.Quantidade * V.ValorUnitario) AS Total
FROM Clientes C
JOIN Vendas V ON C.ID = V.ID_Cliente
JOIN Produtos P ON P.ID = V.ID_Produto;

No Módulo 3, o conceito de Transação é elevado. Não é apenas BEGIN/COMMIT, é sobre garantir as propriedades ACID (Atomicidade, Consistência, Isolamento e Durabilidade).

💻 3. Elementos de Programação (Lógica Pura)

Aqui transformamos o SQL em uma linguagem de script poderosa.

Variáveis e Escopo

Variáveis existem apenas durante a execução do lote (batch). Elas começam com @.

DECLARE @MaiorPreco MONEY;
SET @MaiorPreco = (SELECT MAX(Preco) FROM Produtos);
PRINT 'O produto mais caro custa: ' + CAST(@MaiorPreco AS VARCHAR);

Estruturas de Decisão (IF / ELSE)

Diferente do IF de linguagens como C#, no SQL ele é usado para controlar o fluxo de dados.

IF (SELECT COUNT(*) FROM Vendas WHERE DATEPART(dw, GETDATE()) = 1) > 100
    PRINT 'Meta de domingo atingida!';
ELSE
    PRINT 'Ainda abaixo da meta.';

⚙️ 4. Stored Procedures vs Functions: Detalhamento Técnico

O Módulo 3 foca na diferença crucial de performance e uso entre esses dois objetos [67-70].

Critério Stored Procedures EXEC Scalar/Table Functions SELECT
Objetivo Executar tarefas lógicas, manutenção e processos de negócio. Realizar cálculos ou transformar dados para exibição.
Transações Pode abrir e fechar transações (BEGIN TRAN). Proibido gerenciar transações.
Parâmetros Suporta parâmetros de entrada e OUTPUT. Apenas parâmetros de entrada.
Performance Pré-compilada. Excelente para grandes volumes. Pode ser lenta se usada em muitas linhas (RBAR).
Dica de Ouro: Use Procedures para ações que alteram o banco (Insert/Update). Use Functions para cálculos repetitivos (Converter moeda, formatar CPF) que precisam aparecer no seu SELECT.

🧮 5. Funções de Sistema (O Canivete Suíço)

O material lista funções essenciais para tratar "lixo" nos dados e converter formatos [59-60]:

Texto (Strings):
  • LTRIM/RTRIM: Remove espaços inúteis.
  • REPLACE: Troca caracteres (ex: ponto por vírgula).
  • CHARINDEX: Acha a posição de uma letra.
Datas:
  • DATEADD: Soma dias/meses a uma data.
  • DATEDIFF: Calcula a diferença entre duas datas.
  • EOMONTH: Acha o último dia do mês.

🛠️ 2. As Divisões da Linguagem SQL

O SQL não é uma coisa só. Ele é dividido em sub-linguagens de acordo com a função:

  • DDL (Definição) Cria a estrutura (tabelas, bancos).
    Comandos: CREATE, ALTER, DROP.
  • DML (Manipulação) Mexe nos dados dentro das tabelas.
    Comandos: INSERT, UPDATE, DELETE.
  • DQL (Consulta) Pergunta algo ao banco.
    Comando: SELECT.

📊 3. Tipos de Dados: Como o banco guarda a info

Ao criar uma tabela, você deve dizer exatamente o que cada coluna aceita. Isso economiza memória e evita erros.

🏗️ 4. Prática DDL: Criando Tabelas

Veja como transformar o desenho do Módulo 1 em código real. Atenção às Chaves Primárias (PK) e Estrangeiras (FK):

CREATE DATABASE MinhaLoja;
USE MinhaLoja;

CREATE TABLE Produto (
    ID_Prod INT IDENTITY(1,1) PRIMARY KEY,
    Descricao VARCHAR(100) NOT NULL,
    Preco MONEY
);

CREATE TABLE Venda (
    ID_Venda INT PRIMARY KEY,
    DataVenda DATETIME,
    ID_Prod INT FOREIGN KEY REFERENCES Produto(ID_Prod)
);

IDENTITY(1,1): Faz o banco somar o ID sozinho (1, 2, 3...).

✍️ 5. Prática DML: Inserir e Editar

Agora que a casa está pronta, vamos colocar os móveis (dados):

-- Inserindo um produto
INSERT INTO Produto (Descricao, Preco) 
VALUES ('Teclado Mecânico', 250.00);

-- Aumentando o preço (CUIDADO COM O WHERE!)
UPDATE Produto 
SET Preco = 280.00 
WHERE ID_Prod = 1;

-- Apagando um registro
DELETE FROM Produto WHERE ID_Prod = 1;
⚠️ PERIGO: Nunca esqueça o WHERE no UPDATE ou DELETE, ou você alterará/apagará a tabela INTEIRA!

🔍 6. Consultas Avançadas (SELECT)

A arte de filtrar informação. O Módulo 2 foca em filtros e funções:

  • BETWEEN: Entre dois valores (ex: Preco BETWEEN 10 AND 50).
  • LIKE: Busca por parte do texto (ex: Nome LIKE 'A%' - começa com A).
  • IN: Verifica se o valor está em uma lista (ex: ID IN (1, 5, 10)).
